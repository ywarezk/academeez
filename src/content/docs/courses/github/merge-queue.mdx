title: Merge Queue
description: When multiple developers work on the same repository, CI checks take time to run and must pass against the latest main branch. This creates a race condition where developers wait for their CI checks, only to find that another developer merged their PR first, requiring them to update their branch and restart the CI process all over again. To solve this PR race condition, we will learn how to create a merge queue. GitHub Merge Queue automatically queues approved pull requests and ensures they pass CI checks against an up-to-date main branch before merging, eliminating the race condition by processing PRs sequentially.
